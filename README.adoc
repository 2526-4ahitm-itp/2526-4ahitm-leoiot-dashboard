= LeoIoT Dashboard
:icons: font
:source-highlighter: highlightjs
:toc: left
:toclevels: 2

IoT sensor monitoring dashboard for building management. Tracks temperature and CO2 levels across multiple rooms in real-time.

== Quick Start

[source,bash]
----
# Start all services
docker compose up

# Or start in background
docker compose up -d
----

== Components

* **Mosquitto** - MQTT broker for sensor data
* **InfluxDB 2.7** - Time-series database
* **Telegraf** - MQTT to InfluxDB pipeline
* **Grafana** - Analytics dashboards
* **Quarkus Backend** - Sensor data generator
* **Frontend** - 3D building explorer (Three.js)
* **Dashboard-v2** - Modern sensor dashboard (Chart.js)
* **Fake Sensors** - Simulated sensor data for testing

== Deployment (School VM)

The project is hosted on `vm23.htl-leonding.ac.at`.
A **Reverse Proxy (Nginx)** is used to route all traffic through **Port 80**, bypassing school firewall restrictions for internal ports.

* `/` -> 3D Building Explorer (Port 8080)
* `/dashboard/` -> Dashboard-v2 (Port 8081)
* `/influx/` -> InfluxDB API & UI (Port 8086)
* `/ws` -> WebSocket Bridge (Port 8090)

When changes are pushed to GitHub, update the VM with these commands:
[source,bash]
----

git pull
docker compose up -d --build
----

=== Infrastructure Note
The project uses a **System-level Nginx** as a reverse proxy.
* Config location: `/etc/nginx/sites-available/leoiot`
* To reload after config changes: `sudo systemctl reload nginx`

=== Remote Access (from home)
If you need to access internal ports (like 3000 for Grafana or 8086 for InfluxDB UI) from outside the school network, use an SSH tunnel:

[source,bash]
----
ssh -L 3000:localhost:3000 -L 8086:localhost:8086 solmqadmin@vm23.htl-leonding.ac.at
----
Once connected, you can reach the services at `http://localhost:<PORT>`.
