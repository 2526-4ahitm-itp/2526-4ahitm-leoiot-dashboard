= LeoIoT Dashboard
:icons: font
:source-highlighter: highlightjs
:toc: left
:toclevels: 2

IoT sensor monitoring dashboard for building management. Tracks temperature and CO2 levels across multiple rooms in real-time.

== Quick Start

[source,bash]
----
# Start all services
docker compose up

# Or start in background
docker compose up -d
----

== Access Points

[cols="^.^1,^.^3,^.<3", options="header"]
|===
| Service | URL | Description
| *3D Building View* | http://localhost:8080 | Interactive 3D building with room temperatures
| *Sensor Dashboard* | http://localhost:8081 | Clean dashboard with charts and sensor data
| *Grafana* | http://localhost:3000 | Advanced analytics and visualizations
| *InfluxDB* | http://localhost:8086 | Time-series database UI
|===

== Architecture

----
+-------------+     +----------+     +-------------+
|   Sensors   |---->|  MQTT    |---->|  Telegraf   |
| (Real/Fake) |     | Mosquitto|     |             |
+-------------+     +----------+     +------+------+
                                      |
                                      v
+-------------+     +----------+     +-------------+
|  Frontend   |<----|  Quarkus |<----|  InfluxDB   |
|  Dashboard  |     |  Backend |     |  Database   |
+-------------+     +----------+     +-------------+
----

== Components

* **Mosquitto** - MQTT broker for sensor data
* **InfluxDB 2.7** - Time-series database
* **Telegraf** - MQTT to InfluxDB pipeline
* **Grafana** - Analytics dashboards
* **Quarkus Backend** - Sensor data generator
* **Frontend** - 3D building explorer (Three.js)
* **Dashboard-v2** - Modern sensor dashboard (Chart.js)
* **Fake Sensors** - Simulated sensor data for testing

== Development

[source,bash]
----
# View logs
docker compose logs -f dashboard-v2
docker compose logs -f fake-sensors

# Stop all services
docker compose down
----
