=== Component Specification

==== Services

[cols="1,2,2", options="header"]
|===
| Service | Technology | Role
| `fake-sensors` | Node.js | Simulates room sensors; publishes temperature and CO2 to Mosquitto every 10 s.
| `mqtt-ws-bridge` | Node.js (`mqtt` + `ws`) | Subscribes to Mosquitto, caches latest value per room, fans out live updates to WebSocket clients.
| `telegraf` | Telegraf | Consumes MQTT topics and writes measurements to InfluxDB.
| `solax-collector` | Node.js (`axios`) | Polls Solax Cloud REST API every 5 min and writes PV metrics to InfluxDB.
| `influxdb` | InfluxDB v2 | Time-series store for all historical sensor and PV data (bucket: `server_data`).
| `mosquitto` | Mosquitto | MQTT broker; central message bus between sensors and consumers.
| `nginx` | Nginx | Reverse proxy: routes `/ws` → mqtt-ws-bridge, `/influx` → InfluxDB, `/solax` → Solax API.
| `dashboard-v2` | Vite + Chart.js | Sensor dashboard: live table, temperature/CO2 charts, PV view.
| `frontend` (3D Explorer) | Vite + Three.js | 3D building model with heatmap, room info, and animated indoor navigation.
|===

==== MQTT Topic Hierarchy

[cols="1,2,2", options="header"]
|===
| Topic | Format | Description
| `room-temperature` | JSON: `{"room": "ID", "temperature": float}` | Periodic temperature for all simulated rooms (published by fake-sensors).
| `nili3/sensor/<room>_co2/state` | Float (plain text) | CO2 reading for a specific room, e.g. `nili3/sensor/105_co2/state`.
| `nili3/sensor/nili3_co2/state` | Float (plain text) | CO2 for Room 105 from the physical Nili3 sensor.
| `nili3/sensor/nili3_temperature/state` | Float (plain text) | Temperature for Room 105 from the physical Nili3 sensor.
|===

==== InfluxDB Schema (Bucket: `server_data`)

[cols="1,1,2,2", options="header"]
|===
| Measurement | Field | Tags | Description
| `room_temperature` | `temperature` | `room` | Standardized room temperatures written by Telegraf.
| `mqtt_consumer` | `value` | `topic` | Raw MQTT payloads (CO2 and Nili3 temperature) written by Telegraf.
| `solax_stats` | `daily_yield`, `consumption`, `daily_charged`, `daily_discharged`, `daily_imported`, `daily_exported`, `total_yield` | — | PV plant metrics written by solax-collector.
|===

== Functional Requirements

=== Real-time Monitoring
* Users can view the current temperature and CO2 levels of every room in a 3D building model.
* Data is updated instantly via WebSockets when new sensor values arrive.

=== Historical Analysis
* Users can select a specific date or time range (6h, 24h, 7d) to view trends.
* Aggregated averages are shown for rooms and floors.

=== Mobile Accessibility
* The dashboard is fully responsive and optimized for mobile devices (iOS/Android).
* Support for notch/Dynamic Island devices and touch-friendly interaction targets.

== Non-functional Requirements (NFA)
* **Latency:** Real-time updates should reflect in the UI within 500ms of reaching the MQTT broker.
* **Scalability:** The system should handle up to 200 rooms with 10-second update intervals.
* **Availability:** The dashboard and data collectors should restart automatically via Docker Compose.
